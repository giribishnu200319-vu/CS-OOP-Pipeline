GuessGame — Multi-Agent Number Guessing with Named Pipes
A C# console application demonstrating Inter-Process Communication (IPC), multithreading, and async/await by having multiple Agent processes race to guess a secret number generated by a Master process.

What It Does

The Master generates a random number between 1 and 1000
The Master spawns N Agent processes (default: 2)
Each Agent connects to the Master via a Named Pipe and receives the target number
All Agents independently guess random numbers in a loop until one matches
The winning Agent sends a message back to the Master
The Master announces the winner and prints how long each Agent took

How to Run
Requirements

.NET SDK 8.0+
macOS, Windows, or Linux

Run with default 2 agents
bashcd Master
dotnet run
Run with a custom number of agents
bashdotnet run -- 5    # 5 agents
dotnet run -- 10   # 10 agents

You only ever run the Master. It automatically spawns and manages the Agent processes.

Key Concepts Demonstrated
Inter-Process Communication — Named Pipes
The Master and each Agent communicate through a named pipe called guess-pipe. The Master opens one server-side pipe instance per agent, sends the target number, and waits for the guess result.
csharp// Master side
using var pipe = new NamedPipeServerStream(
    "guess-pipe",
    PipeDirection.InOut,
    NamedPipeServerStream.MaxAllowedServerInstances,
    PipeTransmissionMode.Byte,
    PipeOptions.Asynchronous
);
await pipe.WaitForConnectionAsync();

// Agent side
using var pipe = new NamedPipeClientStream(".", "guess-pipe", PipeDirection.InOut);
await pipe.ConnectAsync();
Async/Await — Non-Blocking Concurrency
The Master handles all agent connections concurrently without blocking any thread, using Task.WhenAny to detect the first winner.
csharpvar pipeTasks = new Task<string>[agentCount];
for (int i = 0; i < agentCount; i++)
    pipeTasks[i] = HandleAgentAsync(target, i + 1, timers[i]);

string winner = await Task.WhenAny(pipeTasks)
                          .ContinueWith(t => t.Result.Result);
Process Spawning
The Master uses Process.Start to launch each Agent as a separate OS process, passing the agent's name as a CLI argument.
csharpvar psi = new ProcessStartInfo
{
    FileName = "dotnet",
    Arguments = $"run --project \"{agentProjectPath}\" -- {agentName}",
    UseShellExecute = false,
    RedirectStandardOutput = true,
};
processes[i] = Process.Start(psi)!;
Performance Timing
Each agent's response time is measured with a Stopwatch that starts when the Agent process launches and stops when the pipe receives the agent's message.
csharptimers[i] = Stopwatch.StartNew();
// ... agent runs ...
timer.Stop();
Console.WriteLine($"Agent {i+1}: {timers[i].ElapsedMilliseconds} ms");
CPU Core Assignment (Windows)
On Windows, each Agent process can be pinned to a specific physical CPU core using ProcessorAffinity. On macOS, this is gracefully skipped since the API is not supported.
csharpif (OperatingSystem.IsWindows())
    processes[i].ProcessorAffinity = (IntPtr)(1 << i);
else
    Console.WriteLine($"Agent {i + 1} started (core pinning skipped on macOS)");
